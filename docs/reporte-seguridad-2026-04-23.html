Resumen ejecutivo

Se aplicaron 17 correcciones de seguridad, accesibilidad y SEO identificadas por la auditoría técnica, más refuerzos adicionales detectados durante el proceso. Las vulnerabilidades críticas de cabeceras HTTP, visibilidad SEO y accesibilidad fueron cerradas. El sistema de registro público se reemplazó por un flujo invite-only con tokens criptográficos de un solo uso, reforzado con confirmación obligatoria de email y plantillas de email transaccional branded.

Todos los sitios tienen ahora Content Security Policy estricto, protecciones anti-clickjacking, formularios de autenticación con atributos accesibles y password-manager friendly, masking de datos sensibles en listas públicas, y visibilidad SEO completa (robots, sitemap, Open Graph dinámico).

2. Sistema de registro invite-only

El endpoint /signup fue cerrado al público. Solo acepta registros con token de invitación válido:

  • Tokens criptográficos: 24 bytes aleatorios (gen_random_bytes) en base64 URL-safe — ~192 bits de entropía.
  • Un solo uso: consumo atómico vía RPC consume_invite() con SECURITY DEFINER — previene race conditions.
  • Expiración automática: 14 días desde creación.
  • RLS sin policies directas: tabla invites inaccesible al cliente anónimo; solo las RPCs con permisos controlados operan sobre ella.
  • Validación por token específico: la RPC validate_invite solo devuelve metadata si el token exacto existe — impide enumerar emails invitados.
  • Email confirmation obligatorio: doble candado de acceso antes del primer login.

3. Infraestructura auth robustecida

  • Ruta /auth/callback creada para procesar email confirmations y OAuth callbacks (PKCE + OTP verification).
  • Env var NEXT_PUBLIC_SUPABASE_ANON_KEY corregida (tenía %0A newline que rompía WebSocket Realtime).
  • Site URL de Supabase Auth actualizado a https://cliente.impulsapr.com.
  • 6 templates de email transaccional reescritos con branding Impulsa PR (Esmeralda + dorado).

4. Formularios accesibles y seguros

  • method="post" explícito en login y signup.
  • Atributos autoComplete="email", current-password, new-password.
  • name e id en todos los inputs (password managers).
  • htmlFor en labels.
